### 4.1.1 整体架构概述

### 4.1.2 各层职责说明

#### ① 前端交互层（Presentation Layer）

| 组件 | 技术栈 | 职责 |
|------|--------|------|
| 求职者Web端 | Vue3 + Vite + Element Plus | 简历管理、职位搜索、投递管理、AI面试 |
| 企业HR Web端 | Vue3 + Vite + Element Plus | 岗位发布、简历筛选、人才库管理 |

**核心交互流程**：
```
用户操作 → HTTP请求/API调用 → 实时响应 + WebSocket通知
```

#### ② 业务逻辑层（Business Layer）

| 模块 | 核心功能 | 技术实现 |
|------|----------|----------|
| 用户认证 | 登录注册、JWT认证、角色切换 | Spring Boot + JWT |
| 简历管理 | CRUD操作、AI解析集成 | MyBatis Plus + AI Service |
| 岗位管理 | 发布、编辑、上下架、搜索 | MyBatis Plus |
| 投递管理 | 申请投递、状态流转 | 状态机 + 通知 |
| 通知消息 | 即时推送、未读计数 | 轮询查询 |
| 智能推荐 | 人岗匹配、简历排序 | AI Service + 规则引擎 |
| 面试管理 | AI面试安排、问答记录、评估报告 | AI Service + 讯飞语音 |
| 数据统计 | 投递分析、转化漏斗、报表生成 | 数据聚合 + ECharts |

#### ③ AI算力层（AI Intelligence Layer）

| 引擎 | 功能描述 | 底层模型 |
|------|----------|----------|
| **LLM大模型服务** | 自然语言理解与生成、对话交互 | 智谱GLM-4 |
| **简历解析引擎** | 简历结构化提取、关键信息识别 | LLM + 正则规则 |
| **语义匹配引擎** | 人岗相似度计算、匹配评分 | LLM + 规则引擎 |
| **面试评估引擎** | 智能问答生成、能力评估 | LLM + 讯飞语音 |
| **模型调度服务** | 异步任务、线程池 | Spring Async + ThreadPool |

**AI服务调用流程**：
```
业务请求 → AI服务 → LLM大模型 → 结果解析 → 返回响应
            ↓
        异步线程池 → 回调通知
```

#### ④ 数据存储层（Data Layer）

| 存储类型 | 用途 | 技术选型 |
|----------|------|----------|
| **关系型数据库** | 核心业务数据存储 | MySQL 8.0 |
| **缓存数据库** | 会话管理、热点数据、验证码 | Redis |
| **对象存储** | 简历文件、图片、文档 | 阿里云OSS |

### 4.2.2 实体关系说明

| 关系 | 类型 | 说明 |
|------|------|------|
| User ↔ JobSeeker | 1:1 | 一个用户对应一个求职者档案 |
| User ↔ Company | 1:1 | 一个用户对应一个企业信息 |
| JobSeeker → Education | 1:N | 一个求职者有多个教育经历 |
| JobSeeker → Experience | 1:N | 一个求职者有多个工作经历 |
| JobSeeker → Project | 1:N | 一个求职者有多个项目经历 |
| Company → Position | 1:N | 一个企业发布多个职位 |
| JobSeeker → Application | 1:N | 一个求职者投递多个岗位 |
| Position → Application | 1:N | 一个职位接收多份投递 |
| Application → Notification | 1:N | 一个投递触发多条通知 |
| Application → Interview | 1:1 | 一个投递对应一个面试记录 |

#### 4.3.2.2 匹配评分公式

**综合匹配分数计算公式**：

```
MatchScore = W1 × SkillScore + W2 × ExperienceScore 
           + W3 × EducationScore + W4 × SalaryScore

其中:
- W1 = 0.50  (技能匹配度权重)
- W2 = 0.30  (经验匹配度权重)
- W3 = 0.10  (学历匹配度权重)
- W4 = 0.10  (薪资匹配度权重)

最终得分: 0-100分
```

#### 4.3.2.3 各维度评分细则

**1. 技能匹配度 (Skill Score)**

```python
def calculate_skill_score(seeker_skills, required_skills):
    if len(required_skills) == 0:
        return 100
    
    intersection = set(seeker_skills) & set(required_skills)
    base_score = len(intersection) / len(required_skills) * 100
    
    SkillScore = min(base_score, 100)
    return SkillScore
```

**2. 经验匹配度 (Experience Score)**

```python
def calculate_experience_score(seeker_years, required_years):
    if seeker_years is None:
        return 50
    
    if seeker_years < required_years:
        deficit = required_years - seeker_years
        score = max(100 - deficit * 15, 40)
    elif seeker_years <= required_years + 3:
        score = 100
    else:
        excess = seeker_years - required_years
        score = max(100 - excess * 3, 70)
    
    return score
```

**3. 学历匹配度 (Education Score)**

```python
def calculate_education_score(seeker_level, required_level):
    if seeker_level >= required_level:
        return 100
    else:
        deficit = required_level - seeker_level
        return max(100 - deficit * 20, 30)
```

**4. 薪资匹配度 (Salary Score)**

```python
def calculate_salary_score(seeker_expected, position_min, position_max):
    if seeker_expected is None:
        return 80
    
    if position_min <= seeker_expected <= position_max:
        return 100
    elif seeker_expected < position_min:
        return max(100 - (position_min - seeker_expected) * 5, 50)
    else:
        return max(100 - (seeker_expected - position_max) * 5, 50)
```

### 4.5.2 匹配算法设计

#### 4.5.2.1 综合评分公式

```
综合匹配分数 = 技能匹配率 × 0.5 + 经验匹配率 × 0.3 + 学历匹配率 × 0.1 + 薪资匹配率 × 0.1
```

#### 4.5.2.2 各项匹配率计算

| 匹配维度 | 权重 | 计算方法 |
|----------|------|----------|
| 技能匹配 | 50% | 匹配技能数 / 岗位要求技能数 × 100% |
| 经验匹配 | 30% | 根据工作年限对比计算 |
| 学历匹配 | 10% | 根据学历等级对比计算 |
| 薪资匹配 | 10% | 期望薪资是否在岗位薪资范围内 |
